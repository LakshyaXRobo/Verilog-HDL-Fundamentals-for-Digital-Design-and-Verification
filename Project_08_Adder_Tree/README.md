# Adder Tree (Verilog)## Description* Implements a multi-level **Adder Tree** using Verilog HDL.* Designed using two different Verilog modeling styles:  * Continuous Assignment (`assign`)  * Procedural Assignment (`always @(\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\*)`)* Performs hierarchical addition of four binary inputs.* Generates intermediate sums as well as the final output sum.* Both implementations synthesize to equivalent combinational hardware.\---## Inputs|Signal|Width|Description||-|-:|-||a|4-bit|First input operand||b|4-bit|Second input operand||c|8-bit|Third input operand||d|8-bit|Fourth input operand|\---## Outputs|Signal|Width|Description||-|-:|-||sum1|5-bit|Sum of inputs `a` and `b`||sum2|9-bit|Sum of inputs `c` and `d`||sum3|10-bit|Final sum of `sum1` and `sum2`|\---## Operation### Stage 1* `sum1 = a + b`### Stage 2* `sum2 = c + d`### Stage 3* `sum3 = sum1 + sum2`\---## Implementation### Continuous Assignment* Uses the `assign` keyword.* Implements the adder tree using **Dataflow Modeling**.* Suitable for simple combinational logic.### Procedural Assignment* Uses the `always @(\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\*)` block.* Implements the adder tree using **Behavioral Modeling**.* Suitable for more complex combinational designs.\---## Continuous Assignment vs Procedural Assignment|Feature|Continuous Assignment|Procedural Assignment||-|-|-||Modeling Style|Dataflow|Behavioral||Keyword|`assign`|`always @(\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\*)`||Circuit Type|Combinational|Combinational||Best Suited For|Simple Logic|Complex Logic||Synthesizable|Yes|Yes|\---## Tools Used* Xilinx ISE / Xilinx Vivado\---## Simulation* Separate testbenches
